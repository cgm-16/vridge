# finish-task skill notes

- `finish-task` currently tells Codex to use the raw task title for the PR title.
- Repo convention in `docs/agents/app-developer.md` says PR titles/descriptions should use Korean Conventional Commit style: `type(scope): subject`.
- `finish-task` currently says not to close the linked issue in the PR body.
- `AGENTS.md` says the issue should stay open during review and close on merge/completion, which is compatible with a PR footer like `Closes #<issue>`.
